Classes

      Classes This keyword is simply a syntax sugar on top of the well established prototype system.
      A class is like a blueprint — a description of the object to be created.

      Una clase (Entidad abstracta) define las caracteristicas de un objeto. Contiene los métodos y/o variables.
      Un objeto es la instancia de una clase.

      * Un objeto no es una clase; pero una clase puede ser o no un objeto.

      *[A class is the definition, while that an object is the implementation]
